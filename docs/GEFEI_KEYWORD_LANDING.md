# 上篇：如何判断一个关键词是否能做

## 0. 一句话

**不是问「这个词有没有量」，而是问：**

> 若我**单独注册域名**只为这个词做站，投入（外链/开发/时间）换回的流量与钱，值不值？
> 若只是**老站加内页**满足用户，标准可以宽松很多。

**同一词，两种决策完全不同。**

---

## 1. 三步法（哥飞原步骤）

### 第 1 步：Google 搜前 10

| 看什么 | 怎么用 |
|--------|--------|
| 首页 vs 内页 各几个 | 全是强站首页 → 难；多为内页 → 新站有机会 |
| 若是**首页** | 看**整站**流量（Similarweb / 插件） |
| 若是**内页** | 看**该内页**流量（Similarweb 页面级） |

### 第 2 步：Ahrefs KD

- 看 KD + 预估「进前 10 要多少引用域」
- 看前三页各自流量、外链
- 工具：https://ahrefs.com/keyword-difficulty

**注意：** Ahrefs 说的是进**前 10** 的引用域，**不是进前 3**。哥飞经验：前 3 往往要高一个数量级（例：进前 10 要 51 域 → 前三可能估 200+）。

### 第 3 步：外链成本与粗 ROI

对目标站：

```
粗 ROI 思路 ≈ 页面/整站流量 ÷ 引用域数量
```

假设每条外链都有成本 → 流量小、外链门槛高 = **不划算单独开站**。

### 必看数据清单（原文总结）

1. **关键词搜索量** — Trends / Ads 关键词规划 / Similarweb
2. **搜索结果构成** — Google SERP / Ahrefs KD / Similarweb
3. **网站流量** — AITDK 插件 / Similarweb / Ahrefs
4. **网页流量** — Similarweb 单页
5. **网站外链** — Ahrefs Backlinks

---

## 2. 案例精读（原文 10 词里展开的）

### 案例 A：`aes encryption online` → **不建议单独开站**

| 观察 | 结论 |
|------|------|
| 前 10：少量首页 + 大量内页 | 表面有缝 |
| 第 4、9 名整站量很小（3k / 3 万） | 但打不过的人仍强 |
| 第 1 名：整站 23 万，该页 1.7 万；2.8K 引用域 | 巨头页 |
| KD 示意：51 域进前 10；前三估 200+ | 投入巨大 |
| 赢了也只有不到 2 万月访量级 | **ROI 差** |

**决策：**

- ❌ 单独注册域名只打这个词 — 不值得
- ✅ 你已有密码/安全工具站，用户需要 → **做一个内页** OK（目的是满足用户，不是硬排名；用的人多了可能慢慢有排名）

---

### 案例 B：`stargate ai` → **绝对不要当工具站域名**

| 点 | 说明 |
|----|------|
| 性质 | 新闻热词 / 公司项目名，不是可做的通用产品名 |
| 风险 | 做同名站 ≈ 钓鱼/假官网 |
| 热度 | 可能一阵就凉（除非像 OpenAI 持续产品） |

**决策：**

- ❌ 注册域名做站
- ✅ 科技媒体站可做**专题专栏**，持续报道 → 信息中心

---

### 案例 C：`Playback Speed Calculator` → **适合练手，不适合当赚钱主词**

| 点 | 说明 |
|----|------|
| 需求 | 2h30m 视频 1.5x 播多久、省多少时间 — 用 SERP 理解需求 |
| SERP | 前 10 **只有 1 个首页**，其余内页；KD 极低（约 3 引用域进前 10） |
| Trends | 约 2023-03 起有量，仍算较新但已 ~2 年，量不大 |
| 对比 | `speed time calculator` 量更大但意图糊（数学速度距离）、KD 55、约 103 域 |

**决策：**

- ✅ 练手站：上线 → 外链 → 冲前 10 / 前 3
- ❌ 别指望靠它赚钱（量太小）
- 预期：不立刻出词正常，要加链才可能

**方法启示：** 不懂的词 → **先 Google 打开第 1 名** 理解需求（吃谷歌这碗饭就用好谷歌）。

---

### 案例 D：`webp to png` → **量/需求在，但别纯 SEO 硬刚**

| 点 | 说明 |
|----|------|
| KD 看起来不高 | 前排都是**大流量站** |
| query.domains | 相关域名几乎注满 |
| SERP | 那些挑战域名**没进首页** → 说明很难 |

**决策：**

- ❌ 新域名纯 SEO 只打这词
- ✅ 已有图片工具站 → 加内页
- ✅ 想挑战：做更漂亮、全免费甚至开源 → **先非 SEO 渠道推广**，跟时间交朋友，慢慢从靠后排名爬

---

### 案例 E：`ai face swap` → **蛋糕大可做，但成本要 hold 住**

| 点 | 说明 |
|----|------|
| 趋势 | 23 下半年起；当时估日搜可观（例：GPTs 的 6 倍量级故事） |
| KD ~60 | 约 129 域进前 10；第 1 名 ~304 引用域 |
| 前两名 | 都是首页，整站百万级 |
| 有趣现象 | 引用域 1200 的输给 352 的 — **因完全免费 + 用户行为数据更好** |
| 技术坑 | 对手 Canonical 错误，权重跑到 `undefined.` 子域 |
| sitelinks | 仅 mini sitelinks、未出完整 sitelinks → 哥飞认为**还有机会** |

**决策：**

- 可做，但是：几百～上千外链 + **可能要免费策略** 才能打第 1
- 问自己：**成本 hold 住吗？**

**启示：** 外链不是唯一；**产品策略（免费）+ 用户行为 + 技术正确（canonical）** 都影响排名。

## 3. 判词决策树（精读后提炼）

```
拿到词
  │
  ├─ 是品牌/公司/新闻热词且非通用工具？
  │     └─ 否做域名站；媒体可做专题
  │
  ├─ Google 前10：是否全是不可撼动的大站首页？
  │     └─ 是 + 页面流量也不大 → 单独开站否；老站内页看情况
  │
  ├─ 估算：进前10/前3 的外链成本 vs 能拿到的月流量
  │     └─ 流量小 & 外链门槛高 → 否单独开站
  │
  ├─ 搜索量小 + 竞争大 → 直接放弃（单独站）
  ├─ 搜索量够大 + 竞争大 → 可考虑（蛋糕大，分一小块也够；还能长大）
  ├─ 搜索量小 + 竞争小 → 练手可以，别当提款机
  │
  └─ 你的目的？
        ├─ 练流程/第一站 → 低 KD、SERP 多内页
        ├─ 赚钱主站 → 量趋势向上 + ROI 说得通
        └─ 已有站补功能 → 内页即可，排名可放宽
```

### 金句（原文）

> **关键词搜索量小，但竞争又大 → 完全不考虑。**
> **有一定搜索量，即使竞争大一些也没问题。蛋糕足够大，只分到一小块就够；努力还能吃更大。**

> **单独开站** 和 **老站做内页** 是两套标准。

---

## 4. 和 Ahrefs 的衔接

| Ahrefs | 哥飞实操补强 |
|--------|----------------|
| KD 数字 | 必须看 SERP 首页/内页构成 + 整站量 |
| 进前 10 引用域 | 前 3 要另估，往往更高 |
| 流量潜力 | Similarweb 单页流量更直观 |
| 商业潜力 | +「免费才能赢吗」「会不会假官网」 |
| 意图 | 打开第 1 名自己用一遍工具/读一遍文 |

# 下篇：落地页 V1.0 → V1.5 → V2.0

## 0. 落地页是什么

- Landing Page = 访客「落地」的页
- 流量可来自：SEO / 搜索广告 / 展示广告 / 社媒 / 裂变…
- **不能转化的落地页 = 失败的落地页**
- 转化目标可以是：用工具 / 注册 / 付费 / App 激活…
- 社群语境默认：**承接 SEO 流量的落地页**

---

## 1. 三个版本定义（原文）

| 版本 | 定义 | 时代背景 |
|------|------|----------|
| **V1.0** | 工具页 + 博客页 | 万兴早期出海；竞争不饱和，博客能排 |
| **V1.5** | 工具页 + **独立落地页** + 博客 | 博客不够了，**每个有搜量的词单独落地页** |
| **V2.0** | **精品工具页** = 工具功能 + 落地图文 + 结果列表 | 群里教授的版本；一页完成 |

```
V2.0 精品工具页面
  = 工具页 + 落地页 + 结果展示页（三合一）
```

---

## 2. V1.0 详解

- 桌面软件时代：产品介绍页围绕**软件名/功能**
- 其他需求词靠**大量博客**覆盖 → 内容页拿排名 → 导下载
- 问题：算法变严后，堆内容/买链被打；**仅博客难拿需求词**

---

## 3. V1.5 详解（最大创新）

**为每个有搜索量的关键词专门做一个落地页。**

### 页面结构

**第一屏：**

1. 一句话说清工具是什么、解决什么
2. 截图/视频直观展示
3. **大 CTA** → 去用工具

**第二屏起：**

- 围绕关键词的图文
- 目的 ① 用户更深理解场景
- 目的 ② **爬虫语义理解** 页面满足什么需求

### 为何这样设计

- 优化转化
- 帮搜索引擎把页放进对的词

### 进化中的「半吊子」形态

- 落地页里直接放表单（替代 CTA），但提交后跳到 app 控制台
- 例：Canva feature 页 — 表单在落地页，**没在落地页完成全需求**
- 大量 AI/传统工具站仍停在 V1.5：落地页域名 vs `app.domain.com` 工具页分离

### 为何老外爱 V1.5

- 研发管 `app.domain.com`
- 营销用 Wix/Framer 管 `www`
- 分工清晰但 **SEO 竞争下偏过时**

### 国内错误

- 照抄 V1.5 拆两个域名 — **不应该**（在激烈 SEO 下）

---

## 4. V2.0 详解（精品工具页）

### 三合一

| 块 | 内容 |
|----|------|
| **工具** | 表单入口；可当页完成，或提交后仍尽量少跳转 |
| **落地文案** | **后端渲染**大量文字，爬虫能读 |
| **结果列表** | UGC 或精选生成结果（图/音/视频） |

### 为什么三合一

1. **用户体验** — 落地页就能用，匹配谷歌重体验
2. **行为数据** — 停留↑、交互↑ → 利排名
3. **转化** — 少一次跳转少掉一批人
4. **动态内容** — 结果列表让页持续更新、直观证明效果

### 结果展示机制（参考 MJ / Suno / Sora / 海螺）

1. 生成时选公开/私有，**默认公开**；私有常付费
2. **精选**，不是全量倾倒
3. 算法过滤：版权/政治/色情 → 质量打分 → 小流量试探再放大

### V2.0 仍继承

- **每词一页**（V1.5 优点保留）
- **仍要博客**（覆盖工具页盖不住的长尾、保持更新频率）

版本 = **层层递进，保留上一版优点，改缺点。**

---

## 5. V2.0 雷区：海量低质 UGC

- 新站一上来海量 UGC 页 → 质量差 → 浪费爬取预算 → 可见度被压
- 精品工具站策略见：`za408lc1q9` 小课堂
- 正确：主词精品页 + 可控扩展内页，而非垃圾海量页

---

## 6. 版本对照表（一图）

```
V1.0  博客拿词 ──CTA──► 工具/下载页
V1.5  独立落地页(文案+CTA) ──跳转──► 工具页(可能另一域名)
V2.0  ┌─────────────────────────────────────┐
      │  首屏：工具可用                        │
      │  中部：SEO 图文（SSR）                 │
      │  下部：结果/精选列表 + FAQ             │
      │  同域：博客长尾                        │
      └─────────────────────────────────────┘
```

# 对照 Pikbo（立刻能用）

## 判词

| 词类型 | 建议 |
|--------|------|
| `ai video generator` / `ai image generator` | 蛋糕大但像 face swap：外链+免费成本极高 → **不单独当唯一主词** |
| `figure spin video` / `blind box video maker` / `etsy listing video` | 垂直、意图清 → **适合主攻/内页矩阵** |
| 竞品品牌词 | 不做假官网；可做对比文案在博客 |
| 已有 `/effects/*` | 标准是**内页**，不是每个词新域名 |

## 落地页版本

| 现状 | 版本判断 |
|------|----------|
| `/create` 工具在当页 | 接近 **V2 工具块** |
| 首页/effect 有文案+CTA | 部分 **V1.5** |
| Library/社区结果 | 结果列表有雏形，未系统「精选公开默认」 |
| guides 博客轴 | **保留正确**（V2 仍要博客） |

### Pikbo V2 目标形态（哥飞标准）

```
/create 或 /effects/[slug]
  ├─ 首屏：上传 + 选预设 + 生成（工具）
  ├─ 下方：SSR 长文（场景、步骤、FAQ）— 爬虫可读
  ├─ 结果：示例片 / 用户公开片精选
  └─ 内链：相关 effect / for / guides
```

同一域名，不拆 app.pikbo.ai 与 www 两套（除非工程必要，SEO 上哥飞不推荐为「营销方便」而拆）。

# 两篇合一的总口诀

1. **先 Google，再 Ahrefs，再算外链账。**
2. **开站严、内页宽。**
3. **量小竞争大 → 滚；量大可咬一口。**
4. **落地页要转化；SEO 落地要 SSR 文字。**
5. **V2 = 工具+文案+结果同页；仍保留博客。**
6. **别新站倾倒低质海量页。**
