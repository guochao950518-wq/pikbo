## 1. 对老板搜索结论的校准（同意 / 修正）

### ✅ 同意

1. HF / Yiha 本质是 **多模型创意工作台**（套件 + 工作流 + 积分）。  
2. **结构优先抄** HF + Yiha + OpenArt 的「货架 / Remix / 一致性入口 / 积分」。  
3. **垂直必须看妙呀**；**3D 落地看 Meshy**。  
4. 差异化应落在：**自有玩具身份 + 多规格出片 + 卖货/发帖闭环**（通用站与妙呀都没打透「已有实体 SKU 出片卖货」）。

### ⚠️ 关键修正（避免战略跑偏）

| 搜索建议 | 风险 | Pikbo 正确读法 |
|----------|------|----------------|
| 「产品结构优先抄多模型聚合」 | 假多模型 live = 信任自杀 | **抄 IA 壳**；模型只标 **真实 live Seedance** |
| 「垂直深度抄妙呀 IP 全链路」 | 妙呀是 **从零设计 IP**；我们是 **已有玩具出片** | **互补赛道**；只抄「阶段流水线 UI」 |
| 「Meshy 2D→3D 可生产」 | 3D 是另一家公司的护城河 | **P2/P3 集成或外链**；不重写 Meshy |
| 把 Runway/可灵当主对标 | 质量战争烧钱 | 用其 **出片质量标准** 校准 Lab，不拼全站 |

**妙呀 vs Pikbo（必须写清）：**

| | 妙呀 | Pikbo |
|--|------|-------|
| 起点 | 文字灵感 | **你已拥有的玩具照片** |
| 产物 | 世界观、概念图、三视图、商品效果图、IP 包 | **可发可卖短视频**（转台/拆盒/社媒） |
| 用户 | 创作者 / 想做 IP 的人 | **卖家 + 藏家 + 小店** |
| 引擎叙事 | Agent 团队（艺术总监/IP/商品） | 配方 recipes + job intents |
| 威胁 | 若他们加「效果图→短视频」会靠近我们 | 若我们做「文生角色 IP」会撞他们 + 稀释定位 |

**结论：** 妙呀是 **上游设计台**；Pikbo 是 **下游运营/卖货台**。短期 **不正面硬刚**，吃透「已有 SKU → 出片」更深。

- Agent 分工：艺术总监 · IP 设计师 · 商品总监

| 可学 | 不可抄 / 不做 |
|------|----------------|

**Pikbo 潮玩化翻译（不碰文生 IP 主路径）：**

同构的是 **流水线 + 交付包**，不是 **造角色**。

## 4. 差异化一句话（更新）

> **通用站**卖「随便生成」。  
> **妙呀**卖「从零设计一个潮玩 IP」。  
> **Meshy**卖「变成可打印的 3D」。  
> **Pikbo**卖：「**你已经拥有的那只玩具** → 可上架、可开箱、可发抖音的短视频运营台。」

护城河顺序：

1. **玩具 recipe 质量**（转台/拆盒/货架语言）  
2. **SKU 身份一致**（同一只玩具跨多条片）  
3. **卖家交付包**（三件套 + 画幅）  
4. 垂直 SEO 与信任（demo/live/退分诚实）
