## 1. 老板意图（原话对齐）

> 对标 higgsfield.ai/generate 与 yiha.ai/lego，**做一个潮玩版本的他们**。

| 他们 | 本质 | 潮玩版 Pikbo |
|------|------|----------------|
| **HF `/generate`** | 创意生产工作台：预设 + 参数 + 出片 + 资产 | **`/create` · Generate Toy Studio** |
| **Yiha `/lego`** | 垂直模块化工作流壳 → 同一引擎 | **`/modules` · Toy Modules** |
| 他们整站 | 多模型通用 OS | **只做潮玩/卖货这一刀，更深** |

---

## 2. 不是什么 / 是什么

| ❌ 不是 | ✅ 是 |
|---------|------|
| 第二个 Higgsfield 全套（Cinema/MCP/假多模型） | 玩具图 → 可发可卖短视频的 **Generate 台** |
| LEGO 品牌页或抄 Yiha 片 | **模块化 job 货架**（转台/拆盒/社媒/Seller Pack） |
| 从零造 IP（妙呀赛道） | **已有实体玩具** 的运营出片台 |

**一句话：**

> **Pikbo = designer-toy creative suite：Generate 工作台 + Modules 积木工作流。**  
> 引擎诚实（Seedance live）；壳对标 HF/Yiha；内容与 recipe 全自有。

## 3. 双入口信息架构（对标他们）

```text
HF Generate  ──►  /create  (+ /generate 别名)
Yiha /lego   ──►  /modules  (模块墙 → 深链 Create)
Apps 目录    ──►  /apps     (与 modules 同源 registry)
```

共享内核：

```text
owned toy photo
  → recipe / module / job
  → Seedance generate
  → versions + Library + delivery steps
```

## 5. 五步约束（防膨胀）

1. **需求：** 潮玩版他们 = 工作台 + 模块货架 + 出片，不是模型数。  
2. **删除：** 假 Cinema 主路径、假多模型 live、LEGO 商标内容。  
3. **简化：** 一个引擎、一个 registry（workflows）、两条主入口。  
4. **加速：** 模块一点进 Create 预填；同图换 job。  
5. **自动化：** 后置。
